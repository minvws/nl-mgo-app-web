# i18n (Internationalization)

For the translation of content we use the [Lingui library][1]. Lingui can extract the messages from the source code and generate `.po` files that can be used for translations.

There are a few key rules we apply when adding new translations:

**Always use an `id` for the translation messages**
_Since the translations might be reused across different platforms, it [helps to be explicit on the id][2]._

**The content in the source code is leading for the source locale (`nl`).**
_When extracting the messages from the source code, this content of the source code will always overwrite the content of the source locale file._

-   Editing the source locale (`nl`) messages is done in the source code itself.
-   Editing other translations is be done via the translation files (`locales/{locale}.po`).

## Updating the messages

The translation files **should always be kept up-to-date** with the messages in the source code. This is also enforced by a check in the CI pipline.

Updating the messages is done by running:

```
pnpm run messages:extract
```
