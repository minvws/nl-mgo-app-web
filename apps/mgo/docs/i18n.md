# i18n (Internationalization)

For the translation of content we use the [Format.JS / React-intl library][1]. We don't manage the translations inside this repository. Instead, all the content is managed using a [lokalise][2] account.

## Updating the messages

For any changes to be made to the copy, you will need to do so in the [lokalise][2] UI. After you have made the neccessary changes there you can download the latest translations using the npm `messages:download` script.

For this to work however you will need to provide the `LOKALISE_API_TOKEN` and `LOKALISE_PROJECT_ID` environment variables. Either by adding them yourself, or by creating a `.env` file in the root of the `mgo` directory. You can copy the `.env.example` and fill in the required values.

There is a script available for updating all the messages. This script will download the latest translations from lokalise and [pre-compiles them for optimised parsing][3] by `react-intl`.

```
pnpm run messages:update
```
